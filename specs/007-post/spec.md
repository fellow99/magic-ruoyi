# 岗位管理模块规格文档（007-post/spec.md）

> magic-ruoyi 岗位管理模块。定义岗位信息的增删改查、部门关联与岗位选择功能。
>
> 版本: 1.0.0 | 日期: 2026-04-29

---

## 1. 模块概述

岗位管理模块负责维护组织内部的岗位信息，支持按部门树筛选、岗位 CRUD 操作、岗位下拉选择等功能。岗位与部门关联，每个岗位归属于特定部门。

### 1.1 技术栈

| 组件 | 技术选型 |
|------|----------|
| 后端框架 | Spring Boot + MyBatis-Plus |
| 前端框架 | Vue 3.5 + Element Plus |
| 状态管理 | Pinia |
| 数据校验 | Element Plus Form Validation |
| 权限控制 | v-hasPermi 指令 |

### 1.2 核心功能

- 岗位列表查询（支持按部门树、岗位编码、岗位名称、类别编码、状态筛选）
- 岗位新增/修改/删除
- 岗位批量操作（批量删除）
- 岗位数据导出（Excel）
- 岗位下拉选择（optionselect）
- 部门树形筛选（左侧部门树 + 顶部部门下拉）

---

## 2. 功能规格

### 2.1 岗位列表

**查询条件**:

| 条件 | 类型 | 说明 |
|------|------|------|
| 部门树节点点击 | 左侧树形组件 | 点击部门树节点，按该部门及其子部门筛选岗位 |
| 岗位编码 | 文本输入 | 模糊匹配 postCode |
| 类别编码 | 文本输入 | 模糊匹配 postCategory |
| 岗位名称 | 文本输入 | 模糊匹配 postName |
| 部门 | 树形下拉选择 | 选择特定部门筛选 |
| 状态 | 下拉选择 | 正常/停用（字典 sys_normal_disable） |

**列表字段**:

| 字段 | 说明 |
|------|------|
| 岗位编号 | postId（隐藏列） |
| 岗位编码 | postCode |
| 类别编码 | postCategory |
| 岗位名称 | postName |
| 部门 | deptName |
| 排序 | postSort |
| 状态 | status（字典标签展示） |
| 创建时间 | createTime |

**操作**: 修改、删除（行内操作按钮）

### 2.2 岗位新增/修改

**表单字段**:

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 岗位名称 | 文本输入 | 是 | - | 岗位显示名称 |
| 部门 | 树形下拉选择 | 是 | - | 岗位所属部门 |
| 岗位编码 | 文本输入 | 是 | - | 岗位唯一编码 |
| 类别编码 | 文本输入 | 否 | - | 岗位分类标识 |
| 岗位顺序 | 数字输入 | 是 | 0 | 排序值，越小越靠前 |
| 岗位状态 | 单选按钮组 | 否 | 0（正常） | 正常/停用 |
| 备注 | 文本域 | 否 | - | 补充说明 |

**校验规则**:

- 岗位名称不能为空
- 岗位编码不能为空
- 部门不能为空
- 岗位顺序不能为空

### 2.3 岗位删除

- 支持单条删除和批量删除
- 删除前弹出确认对话框
- 删除后自动刷新列表

### 2.4 岗位导出

- 按当前查询条件导出 Excel 文件
- 文件名格式: `post_{timestamp}.xlsx`

### 2.5 岗位下拉选择

- 接口: `GET /system/post/optionselect`
- 支持按 deptId 和 postIds 过滤
- 用于用户管理等模块的岗位选择场景

## 3. 权限控制

| 权限标识 | 功能 | 按钮/操作 |
|----------|------|-----------|
| `system:post:list` | 查看岗位列表 | 页面访问 |
| `system:post:query` | 查看岗位详情 | 查看详情 |
| `system:post:add` | 新增岗位 | 新增按钮 |
| `system:post:edit` | 修改岗位 | 修改按钮 |
| `system:post:remove` | 删除岗位 | 删除按钮 |
| `system:post:export` | 导出岗位 | 导出按钮 |

---

## 4. 数据模型

### 4.1 核心实体: SysPost

| 字段 | 类型 | 说明 |
|------|------|------|
| postId | Long | 岗位 ID（主键） |
| deptId | Long | 部门 ID |
| postCode | String | 岗位编码 |
| postName | String | 岗位名称 |
| postCategory | String | 类别编码 |
| deptName | String | 部门名称（关联查询） |
| postSort | Integer | 排序 |
| status | String | 状态（0=正常, 1=停用） |
| remark | String | 备注 |
| createTime | Date | 创建时间 |
| updateTime | Date | 更新时间 |

### 4.2 关联关系

- SysPost N-1 SysDept: 每个岗位归属于一个部门
- SysUser N-M SysPost: 通过 sys_user_post 关联表
