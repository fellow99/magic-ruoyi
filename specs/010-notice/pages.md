### 2.2 组件结构

| 元素 | 类型 | 说明 |
|------|------|------|
| 公告标题输入 | `<el-input>` | 模糊搜索 |
| 操作人员输入 | `<el-input>` | 按创建者筛选 |
| 类型下拉 | `<el-select>` | 使用 sys_notice_type 字典 |
| 新增按钮 | `<el-button>` | 权限: `system:notice:add` |
| 修改按钮 | `<el-button>` | 权限: `system:notice:edit` |
| 删除按钮 | `<el-button>` | 权限: `system:notice:remove` |
| 数据表格 | `<el-table>` | 支持多选 |
| 公告类型展示 | `<dict-tag>` | 使用 sys_notice_type 字典 |
| 状态展示 | `<dict-tag>` | 使用 sys_notice_status 字典 |

### 2.3 表单数据模型

```typescript
interface NoticeForm {
  noticeId: number | string | undefined;
  noticeTitle: string;
  noticeType: string;
  noticeContent: string;
  status: string;
  remark: string;
  createByName: string;
}

interface NoticeQuery extends PageQuery {
  noticeTitle: string;
  createByName: string;
  status: string;
  noticeType: string;
}
```

### 2.4 表单校验规则

| 字段 | 规则 | 触发时机 | 错误提示 |
|------|------|----------|----------|
| noticeTitle | 必填 | blur | 公告标题不能为空 |
| noticeType | 必填 | change | 公告类型不能为空 |

```
onMounted()
  └── getList()  → 获取公告列表
```

### 2.7 核心交互逻辑

#### 2.7.1 提交表单

```
submitForm()
  ├── 表单校验
  ├── 若有 noticeId → updateNotice(form)
  ├── 若无 noticeId → addNotice(form)
  ├── 提示"操作成功"
  ├── 关闭对话框
  └── 刷新列表
```

#### 2.7.2 删除操作

```
handleDelete(row?)
  ├── 确认对话框: "是否确认删除公告编号为'{noticeIds}'的数据项？"
  ├── 调用 delNotice(noticeIds)
  ├── 刷新列表
  └── 提示"删除成功"
```

### 2.8 使用的字典

| 字典 | 用途 |
|------|------|
| sys_notice_type | 公告类型（通知/公告） |
| sys_notice_status | 公告状态（正常/关闭） |
