# 整体规格文档（Overall Specification）

> magic-ruoyi（魔法积木）. 企业级应用快速开发平台。
> 本文档以技术无关的方式描述系统功能需求与非功能需求。

---

## 1. 系统目的

magic-ruoyi 是一个面向企业管理系统快速交付的平台。它将成熟的企业级开发框架与动态接口引擎融合为一体，提供账户权限、多租户、组织管理、系统监控、工作流编排、代码生成和零代码 API 开发能力。团队可在统一后台中快速搭建业务功能，无需从零构建基础设施。

---

## 2. 目标用户

| 角色 | 职责 |
|------|------|
| 平台管理员 | 管理平台全局配置、租户、系统参数和基础设施 |
| 租户管理员 | 管理本租户内的用户、角色、菜单、部门和业务数据 |
| 业务开发者 | 通过代码生成或可视化脚本编辑器快速创建业务接口和页面 |
| 运维人员 | 监控系统运行状态、查看日志、管理缓存和定时任务 |
| 业务审批人员 | 发起、审批、退回和查看业务流程 |
| 终端用户 | 登录系统使用业务功能 |

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 租户 | 使用系统的独立组织或企业，拥有独立的数据和配置 |
| 动态 API | 通过可视化脚本编辑器创建、无需编译即可生效的 HTTP 接口 |
| 静态接口 | 通过传统编码方式定义、编译后部署的 HTTP 接口 |
| 数据权限 | 控制用户可访问的数据行范围（如仅本部门、仅本人） |
| 流程实例 | 一次具体的业务流程执行过程 |
| 代码生成 | 根据数据库表结构自动生成前后端 CRUD 代码的能力 |
| 人机验证 | 登录时要求用户完成的图形验证，用于防止自动化攻击 |
