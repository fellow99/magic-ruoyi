# 003-User 用户管理模块 - 功能规格

> 本文档定义 magic-ruoyi 用户管理模块的功能需求、业务规则和非功能约束。
> 版本: 1.0.0 | 最后更新: 2026-04-29

---

## 1. 模块概述

用户管理模块是系统管理子系统的核心模块，提供用户账号的全生命周期管理能力。租户管理员通过该模块创建、维护、停用和删除用户账号，为账号分配角色和岗位，管理用户状态，以及批量导入导出用户数据。普通用户可通过个人中心维护个人信息、修改密码和更新头像。

---

## 2. 功能范围

### 2.1 用户 CRUD 管理

| 功能 | 说明 |
|------|------|
| 新增用户 | 填写用户名称、昵称、密码、部门、岗位、角色、手机号、邮箱、性别、状态和备注 |
| 查询用户 | 支持按用户名称、昵称、手机号、状态、部门、角色和时间范围筛选，分页展示 |
| 修改用户 | 编辑用户基本信息、岗位和角色。用户编辑自己时不可修改部门、岗位和角色 |
| 删除用户 | 支持单条删除和批量删除，超级管理员（userId=1）不可删除 |

### 2.2 用户角色分配

| 功能 | 说明 |
|------|------|
| 分配角色 | 在独立页面为用户勾选一个或多个角色，提交后即时生效 |
| 角色过滤 | 仅显示状态为正常的角色，停用角色不可勾选 |
| 权限约束 | 需要 `system:user:edit` 权限 |

### 2.3 用户岗位分配

| 功能 | 说明 |
|------|------|
| 岗位选择 | 在新增/编辑用户时通过多选下拉框选择岗位 |
| 岗位过滤 | 仅显示状态为正常的岗位，停用岗位不可选 |
| 部门联动 | 选择部门后自动加载该部门下的岗位列表 |

### 2.4 用户状态管理

| 功能 | 说明 |
|------|------|
| 状态切换 | 列表页通过开关组件切换用户状态（正常/停用） |
| 确认机制 | 切换前弹出确认对话框，防止误操作 |
| 失败回滚 | 接口调用失败时自动恢复开关状态 |

### 2.5 密码重置

| 功能 | 说明 |
|------|------|
| 管理员重置 | 管理员可为任意用户重置密码，新密码通过弹窗输入 |
| 密码规则 | 长度 5-20 字符，不可包含 `< > " ' \|` 等非法字符 |
| 加密传输 | 密码通过 RSA 加密传输（`isEncrypt: true`） |
| 防重复提交 | 重置密码接口禁用重复提交保护（`repeatSubmit: false`） |

### 2.6 用户导入导出

| 功能 | 说明 |
|------|------|
| 导出用户 | 按当前查询条件导出 Excel 文件 |
| 导入用户 | 上传 Excel 文件批量导入用户，支持选择是否更新已存在数据 |
| 模板下载 | 提供标准 Excel 模板供用户填写 |
| 导入结果 | 导入完成后弹窗显示成功/失败详情 |

### 2.7 个人中心

| 功能 | 说明 |
|------|------|
| 基本信息 | 查看个人信息卡片，包括用户名、手机号、邮箱、部门、角色、创建时间 |
| 修改资料 | 修改昵称、手机号、邮箱和性别 |
| 修改密码 | 输入旧密码、新密码和确认密码，校验通过后更新 |
| 头像上传 | 通过图片裁剪组件上传头像，支持缩放、旋转和预览 |
| 第三方应用 | 查看已绑定的第三方社交账号 |
| 在线设备 | 查看当前账号的在线设备列表 |

## 3. 业务规则

### 3.1 用户名称规则

- 长度 2-20 字符
- 全局唯一，不可重复
- 不可包含 `< > " ' \|` 等非法字符

### 3.2 密码规则

- 新增用户时密码长度 5-20 字符
- 个人中心修改密码时长度 6-20 字符
- 不可包含 `< > " ' \|` 等非法字符
- 新密码与确认密码必须一致
- 默认初始密码通过系统参数 `sys.user.initPassword` 配置

### 3.3 手机号规则

- 必须符合中国大陆手机号格式：`1[3456789][0-9]\d{8}`
- 全局唯一，不可重复

### 3.4 邮箱规则

- 必须符合标准邮箱格式
- 全局唯一，不可重复

### 3.5 超级管理员保护

- userId=1 的超级管理员不可被删除
- 超级管理员不可被重置密码
- 超级管理员不可被修改状态
- 超级管理员在列表中不显示删除、重置密码和分配角色按钮

### 3.6 自我编辑限制

- 用户编辑自己的账号时，不可修改归属部门、岗位和角色
- 提交时自动清空 `roleIds`、`deptId` 和 `postIds` 字段

### 3.7 多租户隔离

- 用户数据按 `tenant_id` 隔离
- 查询、新增、修改、删除操作自动应用租户过滤
- 默认租户 ID 为 `000000`

### 3.8 部门树联动

- 用户管理页面左侧展示部门树
- 点击部门节点后，列表仅显示该部门及其子部门下的用户
- 部门树支持名称搜索过滤

## 4. 权限标识

| 权限标识 | 说明 | 对应操作 |
|----------|------|----------|
| `system:user:list` | 用户查询 | 查看用户列表 |
| `system:user:query` | 用户详情 | 查看用户详情 |
| `system:user:add` | 用户新增 | 新增用户 |
| `system:user:edit` | 用户修改 | 修改用户、分配角色 |
| `system:user:remove` | 用户删除 | 删除用户 |
| `system:user:export` | 用户导出 | 导出用户数据 |
| `system:user:import` | 用户导入 | 导入用户数据 |
| `system:user:resetPwd` | 重置密码 | 重置用户密码 |

---

## 5. 非功能需求

### 5.1 性能要求

- 列表查询响应时间 P95 < 500ms
- 列表默认分页大小 10，支持自定义
- 导出操作不阻塞主线程，使用下载流方式

### 5.2 安全要求

- 密码字段通过 RSA 加密传输
- 重置密码接口禁用防重复提交（允许管理员多次重置）
- 所有写操作记录审计日志
- 用户输入在前端展示时进行 XSS 转义

### 5.3 可用性要求

- 列表支持列显隐配置，用户可自定义显示哪些列
- 部门树默认全部展开
- 导入失败时提供详细的错误信息
