## 1. 接口管理场景

### US-001 创建动态接口

**作为** 开发人员
**我希望** 通过 Web 界面创建 HTTP 接口
**以便** 快速实现业务逻辑而无需编译部署

**验收标准**:
- 支持定义请求路径和请求方法
- 支持编写 MagicScript 脚本
- 支持配置请求参数和响应格式
- 保存后接口立即可用

**优先级**: 必须

---

### US-002 测试接口

**作为** 开发人员
**我希望** 在线测试已创建的接口
**以便** 验证接口逻辑是否正确

**验收标准**:
- 支持发送 HTTP 请求
- 显示响应头和响应体
- 显示执行日志
- 显示接口执行时间

**优先级**: 必须

---

### US-003 发布/下线接口

**作为** 开发人员
**我希望** 控制接口的发布状态
**以便** 管理接口的可用性

**验收标准**:
- 发布后接口对外可用
- 下线后接口返回 404
- 状态变更即时生效

**优先级**: 必须

---

### US-004 导入/导出接口

**作为** 开发人员
**我希望** 导入导出接口配置
**以便** 在不同环境间迁移接口

**验收标准**:
- 支持导出接口配置为文件
- 支持从文件导入接口配置
- 导入时处理冲突

**优先级**: 应该

---
