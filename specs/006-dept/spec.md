# 部门管理模块规格文档 (spec.md)

> magic-ruoyi 部门管理模块。负责部门的增删改查、树形组织机构管理和数据权限关联。
>
> 版本: 1.0.0 | 最后更新: 2026-04-29

---

## 1. 模块概述

部门管理是系统组织架构的基础模块。部门以树形结构组织，支持多级嵌套，是用户归属和数据权限范围的核心载体。

### 1.1 模块定位

```
根机构 (dept_id=100)
  ├── 部门 A ──▶ 子部门 A1
  │              └── 子部门 A2
  └── 部门 B ──▶ 子部门 B1

用户 ──(dept_id)──▶ 所属部门
角色 ──(data_scope)──▶ 部门数据权限范围
```

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 部门CRUD | 部门的创建、查询、修改、删除 |
| 树形组织管理 | 以树形结构展示和操作部门层级关系 |
| 数据权限关联 | 角色的数据权限范围基于部门进行配置 |
| 负责人管理 | 为部门指定负责人（关联用户） |

### 1.3 与其他模块的关系

| 关联模块 | 关系说明 |
|----------|----------|
| 用户管理 | 用户通过 dept_id 关联所属部门，一个用户属于一个部门 |
| 角色管理 | 角色的数据权限范围（data_scope）基于部门树进行配置 |
| 岗位管理 | 岗位通过 dept_id 关联所属部门 |
| 租户管理 | 部门属于特定租户，租户间部门数据隔离 |

## 3. 业务规则

### 3.1 部门树形结构规则

- 部门通过 parent_id 自引用形成树形结构
- 根部门的 parent_id = 0
- ancestors 字段存储从根到当前部门的完整路径，如 "0,100,102"
- 修改部门的上级部门时，需递归更新该部门及所有子孙部门的 ancestors 字段

### 3.2 删除约束

- 存在子部门的部门不可删除
- 存在关联用户的部门不可删除（需先转移用户到其他部门）
- 删除操作为逻辑删除（del_flag = '1'）

### 3.3 唯一性约束

- 同一父部门下，部门名称不可重复
- 部门类别编码（dept_category）用于标识部门类型，非唯一

### 3.4 多租户隔离

- 部门数据按 tenant_id 隔离
- 不同租户可以创建同名部门
- 部门树在租户内独立维护

### 3.5 负责人规则

- 负责人必须是该部门的用户（通过 sys_user.dept_id 关联）
- 负责人字段存储用户ID，非用户名称
- 负责人可为空

## 4. 非功能需求

| 编号 | 需求描述 |
|------|----------|
| NFR-DEPT-001 | 部门树加载响应时间应低于 500ms（P95） |
| NFR-DEPT-002 | 部门树节点数量在合理范围内（建议不超过500节点）时，前端渲染应流畅 |
| NFR-DEPT-003 | 修改部门层级关系时，ancestors 更新操作应具备事务一致性 |
| NFR-DEPT-004 | 删除部门前必须完成子部门和关联用户的校验 |
