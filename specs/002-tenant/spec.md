# 租户管理模块规格文档（Tenant Specification）

> magic-ruoyi 租户管理模块。平台管理员通过本模块管理多租户生命周期、套餐分配和数据同步。
>
> 版本: 1.0.0 | 最后更新: 2026-04-29

---

## 1. 模块概述

租户管理模块为平台管理员提供多租户管理能力，包括租户的增删改查、套餐分配、状态控制、过期管理和数据同步。系统采用行级逻辑隔离模式，所有租户共享数据库和 Schema，通过 `tenant_id` 字段实现数据隔离。

### 1.1 模块范围

| 功能域 | 包含内容 |
|--------|----------|
| 租户管理 | 租户 CRUD、状态切换、过期管理、动态切换 |
| 租户套餐 | 套餐 CRUD、菜单权限配置、状态控制 |
| 数据同步 | 套餐同步、字典同步、参数同步 |

### 1.2 模块边界

**本模块负责**:
- 租户信息的维护
- 租户套餐的定义和菜单权限分配
- 租户状态的启用/停用
- 租户过期时间的设置和管理
- 平台级数据向租户的同步

**本模块不负责**:
- 租户内用户管理（由用户管理模块负责）
- 租户内角色权限管理（由角色管理模块负责）
- 租户登录认证（由认证模块负责）

## 2. 功能需求

### 2.1 租户管理

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| TEN-001 | 系统必须支持租户的创建，创建时自动生成 tenant_id 并初始化租户管理员账号 | 必须 |
| TEN-002 | 系统必须支持租户信息的编辑，包括企业名称、联系人、联系电话、地址、简介、域名等 | 必须 |
| TEN-003 | 系统必须支持租户的逻辑删除，删除后数据保留但标记为已删除 | 必须 |
| TEN-004 | 系统必须支持租户列表的分页查询，支持按租户编号、联系人、联系电话、企业名称搜索 | 必须 |
| TEN-005 | 系统必须支持租户详情的查看 | 必须 |
| TEN-006 | 系统必须支持租户状态的切换（正常/停用），停用后该租户用户无法登录 | 必须 |
| TEN-007 | 系统必须支持为租户分配套餐，套餐决定该租户可访问的菜单范围 | 必须 |
| TEN-008 | 系统必须支持设置租户的过期时间，到期后自动停用 | 应该 |
| TEN-009 | 系统必须支持设置租户的用户数量上限，-1 表示不限制 | 应该 |
| TEN-010 | 系统必须支持为租户绑定独立域名 | 可以 |
| TEN-011 | 系统必须支持租户数据的导出（Excel 格式） | 应该 |

### 2.2 租户套餐管理

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| PKG-001 | 系统必须支持租户套餐的创建，包括套餐名称和关联菜单 | 必须 |
| PKG-002 | 系统必须支持租户套餐的编辑，可修改套餐名称和关联菜单 | 必须 |
| PKG-003 | 系统必须支持租户套餐的逻辑删除 | 必须 |
| PKG-004 | 系统必须支持租户套餐列表的分页查询，支持按套餐名称搜索 | 必须 |
| PKG-005 | 系统必须支持租户套餐状态的切换（正常/停用） | 必须 |
| PKG-006 | 系统必须支持通过树形控件选择套餐关联的菜单 | 必须 |
| PKG-007 | 系统必须支持菜单树选择的父子联动开关 | 应该 |
| PKG-008 | 系统必须支持套餐菜单的全选/全不选和展开/折叠操作 | 应该 |

### 2.3 数据同步

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| SYNC-001 | 系统必须支持将指定租户的套餐同步到最新配置 | 必须 |
| SYNC-002 | 系统必须支持将平台级字典同步到所有租户 | 必须 |
| SYNC-003 | 系统必须支持将平台级参数配置同步到所有租户 | 必须 |

### 2.4 动态租户切换

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| DYN-001 | 系统必须支持平台管理员动态切换到指定租户上下文 | 应该 |
| DYN-002 | 系统必须支持清除动态租户切换，恢复默认上下文 | 应该 |

## 3. 非功能需求

### 3.1 性能

| 编号 | 需求描述 |
|------|----------|
| TEN-NFR-001 | 租户列表查询响应时间应低于 500ms（P95） |
| TEN-NFR-002 | 租户数据同步操作应在 10 秒内完成 |

### 3.2 安全

| 编号 | 需求描述 |
|------|----------|
| TEN-NFR-003 | 新增租户接口必须使用 RSA 加密传输敏感信息（用户名、密码） |
| TEN-NFR-004 | 新增租户接口必须防重复提交 |
| TEN-NFR-005 | 租户间数据必须严格隔离，通过 MyBatis-Plus 租户插件自动过滤 |
| TEN-NFR-006 | 租户状态为停用时，该租户下所有用户无法登录系统 |
| TEN-NFR-007 | 租户过期后，系统应自动停用该租户 |

### 3.3 权限

| 编号 | 需求描述 |
|------|----------|
| TEN-NFR-008 | 租户管理操作需要 `system:tenant:*` 权限标识 |
| TEN-NFR-009 | 租户套餐管理操作需要 `system:tenantPackage:*` 权限标识 |
| TEN-NFR-010 | 同步租户字典和参数配置仅限超级管理员（userId = 1）操作 |

## 4. 业务规则

### 4.1 租户编号规则

- `tenant_id` 为 VARCHAR(20) 类型
- 超级租户（主户）固定为 `000000`
- 新租户的 tenant_id 由系统自动生成

### 4.2 租户状态规则

| 状态值 | 含义 | 登录行为 |
|--------|------|----------|
| `0` | 正常 | 允许登录 |
| `1` | 停用 | 禁止登录 |

### 4.3 租户过期规则

- `expire_time` 为空表示永不过期
- `expire_time` 到期后，系统自动将 `status` 设置为 `1`（停用）
- 过期租户的用户在登录时会被拒绝

### 4.4 用户数量限制

- `account_count` 为租户允许创建的最大用户数
- `-1` 表示不限制用户数量
- 创建用户时需校验是否超出限额

### 4.5 套餐菜单权限

- 套餐的 `menu_ids` 存储允许访问的菜单 ID 列表（逗号分隔）
- 用户实际可用菜单 = 租户套餐菜单 AND 角色分配菜单
- `menu_check_strictly` 控制菜单树选择时是否关联显示父子节点

### 4.6 数据同步规则

- 同步套餐：将租户的菜单权限更新为当前套餐配置的菜单列表
- 同步字典：将平台级（主户）字典数据复制到所有租户
- 同步参数：将平台级（主户）参数配置复制到所有租户
