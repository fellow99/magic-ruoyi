# 菜单管理模块规格文档 (spec.md)

> magic-ruoyi 菜单管理模块。负责菜单的增删改查、树形结构管理、权限标识配置和动态路由生成。
>
> 版本: 1.0.0 | 最后更新: 2026-04-29

---

## 1. 模块概述

菜单管理是系统权限体系的基础模块。菜单定义了系统的导航结构和功能权限点，为前端提供动态路由数据，为后端提供权限校验依据。

### 1.1 模块定位

```
角色 ──(N:M)──▶ sys_role_menu ──▶ 菜单/按钮
                                    │
                                    ├── 目录(M) → 导航分组
                                    ├── 菜单(C) → 页面路由
                                    └── 按钮(F) → 操作权限
```

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 菜单CRUD | 菜单的创建、查询、修改、删除 |
| 树形结构管理 | 菜单的父子层级关系维护 |
| 权限标识配置 | 为菜单/按钮配置权限字符串（perms） |
| 动态路由生成 | 为前端提供基于菜单的路由配置数据 |
| 级联删除 | 删除菜单时同时删除其子菜单 |

### 1.3 菜单类型

| 类型 | 标识 | 说明 | 必填字段 |
|------|------|------|----------|
| 目录（M） | M | 导航分组，不包含具体页面 | 菜单名称、排序、图标、路由地址 |
| 菜单（C） | C | 具体页面，对应前端组件 | 菜单名称、排序、图标、路由地址、组件路径 |
| 按钮（F） | F | 操作权限点，不显示在导航中 | 菜单名称、排序、权限标识 |

### 1.4 与其他模块的关系

| 关联模块 | 关系说明 |
|----------|----------|
| 角色管理 | 角色通过 sys_role_menu 关联菜单，决定用户可见页面和按钮 |
| 租户管理 | 租户套餐通过菜单列表限制租户可用功能范围 |
| 前端路由 | 登录后调用 getRouters 获取菜单数据，动态生成前端路由 |

## 2. 功能需求

### 2.1 菜单CRUD

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-MENU-001 | 系统必须支持菜单的树形列表查询，支持按菜单名称、状态筛选 | 必须 |
| FR-MENU-002 | 系统必须支持新增菜单，包含目录、菜单、按钮三种类型 | 必须 |
| FR-MENU-003 | 系统必须支持修改已有菜单的所有属性 | 必须 |
| FR-MENU-004 | 系统必须支持删除单个菜单，有子菜单时不允许直接删除 | 必须 |
| FR-MENU-005 | 系统必须支持级联删除，可同时删除选中的菜单及其所有子菜单 | 必须 |
| FR-MENU-006 | 系统必须支持查询菜单详情 | 必须 |

### 2.2 树形结构管理

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-MENU-007 | 系统必须以树形表格展示菜单，支持展开/折叠子菜单 | 必须 |
| FR-MENU-008 | 系统必须支持懒加载子菜单数据，展开时按需加载 | 必须 |
| FR-MENU-009 | 系统必须提供菜单下拉树结构，用于上级菜单选择 | 必须 |
| FR-MENU-010 | 新增/修改菜单时，必须支持通过树形选择器选择上级菜单 | 必须 |
| FR-MENU-011 | 菜单的显示顺序（order_num）必须支持同级排序 | 必须 |

### 2.3 权限标识配置

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-MENU-012 | 菜单和按钮类型必须支持配置权限标识（perms） | 必须 |
| FR-MENU-013 | 权限标识格式为 `模块:功能:操作`，如 `system:user:list` | 必须 |
| FR-MENU-014 | 目录类型不需要配置权限标识 | 应该 |

### 2.4 动态路由生成

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-MENU-015 | 系统必须提供接口返回当前用户的菜单路由数据 | 必须 |
| FR-MENU-016 | 返回的路由数据必须包含路由路径、组件路径、菜单名称、图标等 | 必须 |
| FR-MENU-017 | 外链菜单（is_frame=0）的路由地址必须以 http(s):// 开头 | 必须 |
| FR-MENU-018 | 隐藏菜单（visible=1）不出现在侧边栏，但路由仍然有效 | 必须 |
| FR-MENU-019 | 停用菜单（status=1）不出现在路由数据中 | 必须 |

### 2.5 菜单属性

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-MENU-020 | 菜单必须支持配置是否外链（is_frame） | 必须 |
| FR-MENU-021 | 菜单类型必须支持配置是否缓存（is_cache） | 必须 |
| FR-MENU-022 | 菜单必须支持配置显示状态（visible）：显示/隐藏 | 必须 |
| FR-MENU-023 | 菜单必须支持配置状态（status）：正常/停用 | 必须 |
| FR-MENU-024 | 隐藏菜单必须支持配置激活路由（remark字段） | 应该 |
| FR-MENU-025 | 菜单必须支持配置路由参数（query_param） | 应该 |

## 3. 业务规则

### 3.1 菜单类型约束

| 菜单类型 | 路由地址 | 组件路径 | 权限标识 | 外链 | 缓存 | 显示状态 |
|----------|----------|----------|----------|------|------|----------|
| 目录（M） | 必填 | 不填 | 不填 | 支持 | 不适用 | 支持 |
| 菜单（C） | 必填 | 必填 | 可选 | 支持 | 支持 | 支持 |
| 按钮（F） | 不填 | 不填 | 必填 | 不适用 | 不适用 | 不适用 |

### 3.2 删除约束

- 单个删除时，如果菜单存在子菜单，则拒绝删除
- 级联删除时，选中菜单及其所有子菜单一并删除
- 级联删除同时清除 sys_role_menu 中的关联记录

### 3.3 路由生成规则

- 仅返回 status = '0'（正常）的菜单
- 目录（M）和菜单（C）参与路由生成，按钮（F）不参与
- 外链菜单（is_frame = '0'）使用完整 URL 作为路由路径
- 内链菜单的路由路径为 path 字段值
- 组件路径默认在 `views` 目录下拼接

### 3.4 权限标识规则

- 权限标识用于后端接口权限校验（如 `@SaCheckPermission('system:user:list')`）
- 权限标识用于前端按钮显示控制（如 `v-hasPermi="['system:menu:add']"`）
- 权限标识格式约定：`模块:功能:操作`

### 3.5 多租户隔离

- 菜单数据为全局共享，不按租户隔离
- 租户通过租户套餐（tenant_package）限制可用菜单范围
- 用户实际可见菜单 = 租户套餐菜单 AND 角色分配菜单

## 4. 非功能需求

| 编号 | 需求描述 |
|------|----------|
| NFR-MENU-001 | 菜单树加载应在 1 秒内完成 |
| NFR-MENU-002 | 懒加载子菜单响应时间应低于 300ms（P95） |
| NFR-MENU-003 | 路由数据接口响应时间应低于 500ms（P95） |
| NFR-MENU-004 | 菜单树形表格应支持大数据量渲染（100+ 节点） |
