# 角色管理模块规格文档 (spec.md)

> magic-ruoyi 角色管理模块。负责角色的增删改查、菜单权限分配、数据权限范围配置、状态管理和用户分配。
>
> 版本: 1.0.0 | 最后更新: 2026-04-29

---

## 1. 模块概述

角色管理是系统权限体系的核心模块。角色作为用户与权限之间的桥梁，通过角色-菜单-按钮的绑定实现功能权限控制，通过角色的数据权限范围实现行级数据隔离。

### 1.1 模块定位

```
用户 ──(N:M)──▶ sys_user_role ──▶ 角色 ──(1:N)──▶ sys_role_menu ──▶ 菜单/按钮
                                              ──▶ sys_role_dept ──▶ 部门（数据权限）
```

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 角色CRUD | 角色的创建、查询、修改、删除 |
| 菜单权限分配 | 为角色勾选可访问的菜单和按钮 |
| 数据权限范围 | 控制角色可查询的数据行范围 |
| 状态管理 | 启用/停用角色，停用后该角色不再生效 |
| 用户分配 | 将用户关联到角色，支持批量操作 |

### 1.3 与其他模块的关系

| 关联模块 | 关系说明 |
|----------|----------|
| 用户管理 | 用户通过 sys_user_role 关联角色，一个用户可拥有多个角色 |
| 菜单管理 | 角色通过 sys_role_menu 关联菜单，决定用户可见页面和按钮 |
| 部门管理 | 数据权限为"自定义"时，通过 sys_role_dept 指定可访问的部门 |
| 租户管理 | 角色属于特定租户，租户间角色数据隔离 |

## 2. 功能需求

### 2.1 角色CRUD

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-ROLE-001 | 系统必须支持角色的分页列表查询，支持按角色名称、权限字符、状态筛选 | 必须 |
| FR-ROLE-002 | 系统必须支持按创建时间范围筛选角色 | 必须 |
| FR-ROLE-003 | 系统必须支持新增角色，包含角色名称、权限字符、显示顺序、状态、菜单权限、备注 | 必须 |
| FR-ROLE-004 | 系统必须支持修改角色基本信息和菜单权限 | 必须 |
| FR-ROLE-005 | 系统必须支持批量删除角色，超级管理员角色（roleId=1）不可删除 | 必须 |
| FR-ROLE-006 | 系统必须支持角色数据导出为Excel | 应该 |
| FR-ROLE-007 | 角色名称在租户内必须唯一 | 必须 |
| FR-ROLE-008 | 权限字符在租户内必须唯一 | 必须 |

### 2.2 菜单权限分配

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-ROLE-009 | 新增/修改角色时，必须提供菜单树供勾选权限 | 必须 |
| FR-ROLE-010 | 菜单树必须支持展开/折叠、全选/全不选操作 | 必须 |
| FR-ROLE-011 | 菜单树必须支持父子联动开关（严格模式/非严格模式） | 必须 |
| FR-ROLE-012 | 修改角色时，必须回显已勾选的菜单节点 | 必须 |
| FR-ROLE-013 | 菜单权限保存时，必须同时保存选中节点和半选中节点 | 必须 |

### 2.3 数据权限范围

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-ROLE-014 | 系统必须支持为角色配置数据权限范围 | 必须 |
| FR-ROLE-015 | 数据权限范围必须包含6种模式：全部数据、自定义数据、本部门数据、本部门及以下数据、仅本人数据、部门及以下或本人数据 | 必须 |
| FR-ROLE-016 | 选择"自定义数据权限"时，必须提供部门树供勾选具体部门 | 必须 |
| FR-ROLE-017 | 部门树必须支持展开/折叠、全选/全不选、父子联动操作 | 必须 |
| FR-ROLE-018 | 切换数据权限范围为非自定义时，必须清空已选的部门 | 必须 |
| FR-ROLE-019 | 数据权限修改后，必须在下次数据查询时生效 | 必须 |

### 2.4 状态管理

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-ROLE-020 | 系统必须支持通过开关快速切换角色状态（启用/停用） | 必须 |
| FR-ROLE-021 | 状态切换前必须弹出确认对话框 | 必须 |
| FR-ROLE-022 | 状态切换失败时，必须回滚开关状态 | 必须 |
| FR-ROLE-023 | 停用的角色不再生效，拥有该角色的用户失去对应权限 | 必须 |

### 2.5 用户分配

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-ROLE-024 | 系统必须支持查看已分配给角色的用户列表 | 必须 |
| FR-ROLE-025 | 已分配用户列表必须支持按用户名称、手机号搜索 | 必须 |
| FR-ROLE-026 | 系统必须支持查看未分配给角色的用户列表 | 必须 |
| FR-ROLE-027 | 系统必须支持从用户选择弹窗中批量添加用户到角色 | 必须 |
| FR-ROLE-028 | 系统必须支持取消单个用户的角色授权 | 必须 |
| FR-ROLE-029 | 系统必须支持批量取消用户的角色授权 | 必须 |

## 3. 业务规则

### 3.1 角色唯一性约束

- 同一租户内，角色名称（role_name）不可重复
- 同一租户内，权限字符（role_key）不可重复
- 权限字符用于代码中的权限校验（如 `@SaCheckRole('admin')`）

### 3.2 超级管理员保护

- roleId = 1 的角色为超级管理员
- 超级管理员不可删除
- 超级管理员不可修改数据权限
- 超级管理员拥有全部权限，不受菜单分配限制

### 3.3 数据权限规则

| data_scope 值 | 含义 | 数据过滤逻辑 |
|---------------|------|-------------|
| 1 | 全部数据权限 | 无额外过滤，可查看全部数据 |
| 2 | 自定义数据权限 | 仅可查看 sys_role_dept 中指定部门的数据 |
| 3 | 本部门数据权限 | 仅可查看用户所属部门的数据 |
| 4 | 本部门及以下数据权限 | 可查看用户所属部门及其所有子部门的数据 |
| 5 | 仅本人数据权限 | 仅可查看用户自己创建的数据 |
| 6 | 部门及以下或本人数据权限 | 可查看用户所属部门及子部门的数据，或用户自己创建的数据 |

### 3.4 菜单权限规则

- 菜单权限通过树形结构勾选，包含目录（M）、菜单（C）、按钮（F）三种类型
- menuCheckStrictly = true 时，勾选父节点自动勾选所有子节点
- menuCheckStrictly = false 时，可独立勾选任意节点
- 保存时需同时记录选中节点和半选中节点的 ID

### 3.5 删除约束

- 删除角色前需检查是否有用户关联该角色
- 已分配用户的角色删除时，需同步清除 sys_user_role 中的关联记录
- 删除操作为逻辑删除（del_flag = '1'）

### 3.6 多租户隔离

- 角色数据按 tenant_id 隔离
- 不同租户可以创建同名角色
- 角色关联的菜单权限受租户套餐限制（用户实际权限 = 租户套餐菜单 AND 角色分配菜单）

## 4. 非功能需求

| 编号 | 需求描述 |
|------|----------|
| NFR-ROLE-001 | 角色列表查询响应时间应低于 500ms（P95） |
| NFR-ROLE-002 | 菜单树加载应在 1 秒内完成 |
| NFR-ROLE-003 | 角色状态切换操作应具备乐观更新和失败回滚能力 |
| NFR-ROLE-004 | 数据权限变更应在下次查询时立即生效，无需重新登录 |
| NFR-ROLE-005 | 角色导出应支持大数据量异步处理 |

## 5. 权限标识

| 权限标识 | 对应操作 | 说明 |
|----------|----------|------|
| system:role:list | 查询角色列表 | 列表页面访问和搜索 |
| system:role:query | 查询角色详情 | 查看单个角色信息 |
| system:role:add | 新增角色 | 创建新角色 |
| system:role:edit | 修改角色 | 编辑角色信息和数据权限 |
| system:role:remove | 删除角色 | 删除角色和取消用户授权 |
| system:role:export | 导出角色 | 导出角色列表为Excel |
